# Projet 1 – Simulation de tickets IT

## Français

### Objectif
Prouver que je sais prendre une demande d’utilisateur, la comprendre, poser les bonnes questions, documenter mes actions et proposer une solution claire.

### Ce que j’ai fait
J’ai utilisé Trello (gratuit) pour créer 5 tickets réalistes. Pour chaque ticket, j’ai noté :
- la demande de l’utilisateur (exactement ce qu’il aurait écrit)
- les questions que je lui aurais posées
- mon diagnostic (ce que je vérifie, mes hypothèses)
- les actions que j’ai faites (commandes, redémarrages, paramètres)
- la solution finale

### Exemple de ticket
**Demande :** « Je n’arrive plus à me connecter à mon VPN. »  
**Questions :** Depuis quand ? As-tu changé ton mot de passe récemment ? Quelle est l’erreur affichée ?  
**Diagnostic :** J’ai vérifié que son compte n’était pas verrouillé dans Active Directory, que le service VPN tournait, que le pare-feu ne bloquait pas le port.  
**Actions :** J’ai réinitialisé son profil VPN, relancé le service, débloqué le port 443 dans Windows Defender.  
**Solution :** Le port était bloqué par une mise à jour. Après déblocage, la connexion fonctionne.

### Compétences démontrées
- Gestion de tickets
- Priorisation (SLA)
- Communication client
- Documentation
